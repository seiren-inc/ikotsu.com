## 3. トップページ構造

Hero → 専門性説明 → サービス一覧 → 診断CTA → 工程紹介 → 料金透明性 → 法人案内 → FAQ抜粋 → 関連サービス導線 → フッター
**トップは説明ページではない。ハブである。**

---

## 4. サービスセクション構造

`/services` 配下：粉骨 / 洗骨 / 出張 / 改葬前処理
**各ページ構成：** 概要 → 料金 → 工程 → 安全性 → FAQ → CTA（構造統一）

---

## 5. ガイドセクション設計

`/guide` 配下カテゴリ：保存 / 法律 / 工程 / トラブル / 法人向け
記事は商用ページへ内部リンク。**孤立禁止。**

---

## 6. 法人セクション設計

`/for-corporate` 構成：法人向けトップ → 課題提起 → 解決策提示 → 料金モデル → 資料DL → 問い合わせ
**法人は独立導線。**

---

## 7. URL設計原則

短い / 英語スラッグ / 意味が明確 / ハイフン区切り
例：`/guide/mold-risk`, `/services/powdered-bone`
**パラメータ型URL禁止。**

---

## 8. パンくず設計

トップ ＞ ガイド ＞ 保存 ＞ カビ
構造化データ対応。SEO強化。

---

## 9. 内部リンク戦略

ガイド → サービス / サービス → 診断 / 法人記事 → 法人ページ / FAQ → 該当ページ
**循環構造を作る。**

**ヘッダー：** サービス / 法人の方へ / ガイド / 会社情報 / お問い合わせ
サービスはプルダウン。
