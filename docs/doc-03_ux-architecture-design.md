# クロスサービスUXアーキテクチャ設計書

**Project:** Seiren Ecosystem UX Master Map
**Goal:** 遺骨.comを起点に清蓮グループ全体へ自然に流すUX設計

---

## 1. UX設計思想

遺骨.comは「処理の場」ではない。**「供養の起点」** である。

UXは以下の流れで設計する：

```
不安 → 整備 → 選択 → 次の供養 → 安心
```

---

## 2. ユーザージャーニーマップ

### フェーズ1：不安

- 遺骨の状態が心配
- カビや湿気が気になる
- どうすればいいか分からない

**導線：** トップ → 状態診断 → 粉骨 or 洗浄

### フェーズ2：整備

- 粉骨／洗浄／取り出し
- 料金シミュレーター
- 無料相談

### フェーズ3：選択

整備後の分岐：

| 選択肢 | 内容                 |
| ------ | -------------------- |
| A      | 散骨したい           |
| B      | 墓じまいをしたい     |
| C      | 新しい墓を探したい   |
| D      | 自宅供養したい       |
| E      | ダイヤモンドにしたい |
| F      | 終活全体を相談したい |

### フェーズ4：移動

各専門サービスへ遷移：

- 散骨クルーズ
- お墓じまいナビ
- お墓探しナビ
- 遺骨ダイヤモンド
- 終活コンシェルジュ

### フェーズ5：安心

- 作業完了通知
- レポート表示
- 次の提案

## 4. クロスサービス導線設計

### 粉骨完了画面

表示：「整えたあとの供養方法を選ぶ」

| カード           | リンク先           |
| ---------------- | ------------------ |
| 海へ還す         | 散骨クルーズ       |
| お墓を整理する   | お墓じまいナビ     |
| 新しいお墓を探す | お墓探しナビ       |
| 形に残す         | ダイヤモンド       |
| 全体を相談       | 終活コンシェルジュ |

### 各サービスページ下部

「遺骨整備がまだの方はこちら」→ 遺骨.comへ戻す

## 5. ナビゲーション設計

ヘッダーに「清蓮グループ」メニュー。クリックでエコシステムマップ表示ページへ。

**エコシステム可視化図（イメージ）：**

```
          散骨クルーズ
               ↑
墓じまいナビ ← 遺骨.com（中央） → お墓探しナビ
               ↓
    ダイヤモンド / 終活コンシェルジュ
```
