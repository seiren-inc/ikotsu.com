# データアーキテクチャ・インテリジェンス設計書

**Project:** Ikotsu.com Data Architecture & Intelligence Design
**Layer:** Data Infrastructure / Strategic Asset Design
**Version:** 1.0
**Purpose:** 遺骨.comを"作業型ビジネス"から"データ資産型ビジネス"へ進化させるための基盤設計

---

## 1. 設計目的

粉骨は単発業務に見えるが、本質は状態データ / 地域データ / 法人発注データ / 価格感応データの蓄積。
**データは最大の参入障壁。**

---

## 2. データカテゴリ設計

| カテゴリ              | 項目例                                                   |
| --------------------- | -------------------------------------------------------- |
| **A：顧客データ**     | 問い合わせ内容 / 地域 / 用途 / 成約有無 / オプション選択 |
| **B：遺骨状態データ** | 湿気レベル / カビ発生率 / 保存期間 / 容器種類            |
| **C：法人データ**     | 寺院名 / 月間件数 / 単価 / 契約状況                      |
| **D：業務データ**     | 処理日数 / 工程時間 / 再処理率                           |

---

## 3. データ設計原則

匿名化前提 / 統計利用優先 / 個人特定回避 / 外部API送信制限
**安全が最優先。**

---

## 4. データ活用戦略

- **4.1 地域別カビ率分析** → 洗骨提案ロジック最適化
- **4.2 保存年数別劣化分析** → 保存ガイド記事作成
- **4.3 法人ボリューム分析** → 割引設計最適化
  **データ→利益。**

---

## 5. 価格戦略への活用

AIが地域×状態×件数を分析し、最適価格帯を提案。将来拡張可能。

---

## 6. SEO強化への活用

蓄積データから：「◯◯県の遺骨カビ発生率」「保存10年後の変化」などオリジナル統計公開。
**被リンク誘発。**

---

## 7. 法人営業への活用

提案資料に：平均処理日数 / 再発防止率 / 湿気対策成功率（数値で説得）

---

## 8. データ構造設計

顧客ID / 案件ID / 法人ID / 処理ID → 紐付け可能構造。将来CRM統合可能。

---

## 9. BI設計

**ダッシュボード：** 月次受注数 / 平均単価 / 法人比率 / オプション率 / CV率（経営可視化）

---

## 10. データ保護設計

アクセス権限管理 / ログ記録 / 削除ポリシー / 保存期間設定
**YMYL領域は厳格。**

---

## 11. 3年ビジョン

粉骨・洗骨分野で最大の実データ保有企業へ。業界レポートを出せる存在に。

---

## 12. 最終目標

遺骨.comを**データドリブン供養基盤**へ。作業ではなく、知見で勝つ。
