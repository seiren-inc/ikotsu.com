# リスク管理・危機対応プレイブック

**Project:** Ikotsu.com Risk Management & Crisis Response Playbook
**Layer:** Strategic Risk Control
**Version:** 1.0
**Purpose:** 遺骨という極めてセンシティブな領域において、炎上・法的問題・誤解・事故発生時に即時対応できる危機管理設計

---

## 1. 前提

遺骨ビジネスは高リスク領域。
**想定リスク：** 価格トラブル / 法的誤解 / 宗教的批判 / 工程事故 / SNS炎上 / 情報漏洩 / 競合からの攻撃
「起きない」前提は危険。起きた時の設計が必要。

---

## 2. リスク分類

- **Level 1：** 軽微クレーム
- **Level 2：** SNS拡散リスク
- **Level 3：** 法的問題
- **Level 4：** ブランド毀損級
  段階対応設計。

---

## 3. Level 1対応

**例：** 価格誤解 / 納期誤解 / 説明不足
**対応：** 即日返信 / 事実説明 / 記録提示 / 冷静対応
サイト表現改善も同時実施。

---

## 4. Level 2対応

**例：** SNSでの誤情報拡散 / 「違法では？」という投稿
**対応：** 事実確認 / 公式声明準備 / 感情的反応禁止 / 法的整理文を提示
透明性を維持。

---

## 5. Level 3対応

**例：** 法律違反疑義 / 行政指導 / 重大クレーム
**対応：** 即時専門家相談 / 社内対応チーム設置 / 記録整理 / 公開対応文書準備
感情論で対応しない。

---

## 6. Level 4対応

**例：** 重大事故 / 遺骨紛失 / 大規模炎上
**対応：** 即時公開説明 / 責任の所在明示 / 再発防止策提示 / 外部専門家関与
**隠蔽は絶対禁止。**

---

## 7. 事前予防設計

料金明確化 / 工程写真保存 / ログ管理 / 二重チェック体制 / 法的表現確認
リスクは未然に潰す。

---

## 8. 炎上防止表現ガイドライン

**禁止：** 断定的法解釈 / 他社批判 / 過度な価格優位表現 / 宗教軽視発言
**使用：** 解説型表現 / 事実ベース / 配慮ある言葉

---

## 9. 社内対応フロー

1. 事実確認 → 2. 記録収集 → 3. 影響範囲特定 → 4. 対応方針決定 → 5. 公開文作成 → 6. 改善実施
   即断即決しない。

---

## 10. Web緊急対応

誤表記即修正 / 該当ページ一時非公開 / 告知文掲載 / FAQ更新
透明性優先。

---

## 11. KPI

重大事故ゼロ / 炎上ゼロ / 法的問題ゼロ / クレーム率低下

---

## 12. 最終目標

トラブルが起きても、信頼を失わない組織。
**リスク管理まで設計された専門機関。**
