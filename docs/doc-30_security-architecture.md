# セキュリティ設計・リスクコントロール設計書

**Project:** Ikotsu.com Security Architecture & Risk Control Design
**Layer:** Functional Design
**Version:** 1.0
**Purpose:** 遺骨関連という極めてセンシティブな領域において、法的・技術的・ブランド的リスクを最小化するためのセキュリティ設計

---

## 1. セキュリティ設計の前提

Ikotsu.comは「遺骨」を扱う。通常のECよりも高い信頼水準が必要。
**守る対象：** 個人情報 / 相談内容 / 遺骨状態情報 / 法人情報 / 価格情報 / ブランド信用
**セキュリティは信頼の土台。**

---

## 2. 技術的セキュリティ設計

### 2.1 通信

SSL必須（HTTPS強制） / HSTS有効化 / TLS1.2以上

### 2.2 フォーム対策

reCAPTCHA v3以上 / CSRFトークン / レート制限 / Bot遮断

### 2.3 API対策

入力値サニタイズ / SQLインジェクション対策 / 型チェック / エラーメッセージ最小化

---

## 3. データ保護設計

**保存データ：** 問い合わせ内容 / 連絡先 / 診断履歴 / 見積履歴
**設計：** 暗号化保存（将来DB拡張時） / 管理画面アクセス制限 / ログ保存 / 定期削除ポリシー
**保管期限：** BtoC問い合わせ 3年 / BtoB問い合わせ 5年

---

## 4. 管理画面セキュリティ

二段階認証 / IP制限（将来） / アクセスログ保存 / 操作履歴保存
**権限レベル：** Admin / Staff / Viewer

---

## 5. 法的リスク管理

**表示必須：** プライバシーポリシー / 特定商取引法表記 / 利用規約 / 免責事項
**注意：** 粉骨に関する法律説明は断定しすぎない。宗教的配慮を明示。

---

## 6. ブランドリスク対策

過度な表現禁止 / 価格誤表示防止 / 誤解を招く比較禁止 / 誇張広告禁止
**炎上回避設計。**

---

## 7. 障害対応設計

**想定リスク：** フォーム送信不能 / API停止 / サーバーダウン / 誤表示
**対策：** 監視設定 / エラーログ通知 / 即時修正フロー / 代替連絡手段表示

---

## 8. バックアップ設計

定期バックアップ / データ復元テスト / 設定ファイル管理

---

## 9. 将来リスク拡張対策

- 法人ポータル導入時のアクセス制御
- 顧客マイページ導入時の認証強化
- PDF自動生成時のデータ保護

---

## 10. KPI

セキュリティ事故ゼロ / スパム率5%未満 / データ漏洩ゼロ

---

## 11. 最終目標

**「安心して相談できる専門機関」**
技術と法務で守られた信頼ブランド。
