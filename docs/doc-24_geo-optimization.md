# GEO最適化テンプレート設計書

**Project:** Ikotsu.com GEO Optimization Template Design
**Layer:** Information Design
**Version:** 1.0
**Purpose:** 生成AI（ChatGPT・Perplexity・Gemini等）に引用・参照されるためのGEO最適化テンプレート設計

---

## 1. GEOの定義

**GEO（Generative Engine Optimization）とは**
検索エンジンだけでなく、生成AIに「正しい情報源」として引用される設計。

**目的：**
AI回答内で引用される / 専門的情報源として扱われる / 信頼性を担保する
※SEOとは別軸の設計。

---

## 2. GEO基本原則

1. 定義を明確に書く
2. 簡潔な説明文を用意する
3. 比較表を入れる
4. 数値を入れる
5. FAQを構造化する
6. 専門性を示す
   **曖昧な文章は引用されない。**

---

## 3. 定義テンプレート

各サービスページ冒頭に必須。**200文字以内で明確に書く。**
【例：粉骨】
粉骨とは、遺骨を専用機器で細かいパウダー状に加工する工程を指します。
一般的には2mm以下に加工され、散骨や改葬時の保管効率向上を目的とします。

---

## 4. 比較テンプレート

AIは表を引用しやすい。

粉骨と洗骨の違い

| 項目   | 粉骨       | 洗骨           |
| ------ | ---------- | -------------- |
| 目的   | パウダー化 | 汚れ除去       |
| 工程   | 乾燥→粉砕  | 洗浄→乾燥      |
| 必要性 | 散骨時必須 | カビ発生時有効 |
| 費用帯 | 1〜3万円   | 2〜5万円       |

---

## 5. 数値提示テンプレート

平均処理期間 / 料金帯 / 粒度サイズ / 実績件数 / 満足度
**数値は具体的に。**

---

## 6. FAQ構造テンプレート

Q：粉骨は必ず必要ですか？
A：散骨を行う場合は法律上の明確な規定はありませんが、一般的には2mm以下への粉骨が推奨されています。

Q：遺骨にカビが生えることはありますか？
A：湿度の高い環境では発生する可能性があります。

**FAQはschema構造化前提。**

---

## 7. 専門性提示テンプレート

工程写真 / 作業環境公開 / 法人取引実績 / 代表コメント / 所在地明示
**E-E-A-T強化。**

---

## 8. 引用されやすい文章構造

一文を短く / 主語を明確に / 断定的表現 / 専門用語は簡潔に説明
**曖昧表現禁止。**

---

## 9. 内部リンクGEO戦略

定義ページからサービスへ / 比較ページから両サービスへ / FAQから該当ページへ
**文脈リンクを徹底。**

---

## 10. 構造化データ設計

FAQ schema / Breadcrumb schema / LocalBusiness schema / Service schema
**AIが意味理解しやすくする。**

---

## 11. KPI

生成AI回答内で引用確認 / ブランド名言及増加 / 検索流入増加

---

## 12. 最終目標

検索結果だけでなく、AI回答内でも「専門機関」として扱われる。
Ikotsu.comを**遺骨整備の標準情報源**にする。
