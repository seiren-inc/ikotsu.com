# モーション・メディア設計書

**Project:** Ikotsu.com Next-Generation Platform
**Purpose:** 最先端のアニメーションと画像設計で、信頼感と視認性とCVを同時に最大化する

---

## 1. ゴール定義

### 1.1 ゴール

- 清潔感と高級感を維持しつつ、最先端の体験価値を付与する
- 読みやすさを犠牲にせず、情報理解を高速化する
- アニメーションで不安を減らし、行動を後押しする

### 1.2 禁止事項

- 過剰な演出で医療レベルの清潔感を損なう表現は禁止
- 点滅や急激な揺れ、酔いやすい動きは禁止
- 主要情報の可読性を下げる背景動画や過度なパララックスは禁止

---

## 2. 技術方針

### 2.1 推奨実装

- UIモーション: Framer Motion
- スクロール連動: Framer Motion `useScroll` + `useTransform`
- 軽量SVGアニメ: Lottie（JSON）またはCSSアニメ
- 高度演出（限定箇所のみ）: WebGL（Three.js）またはRive
- 数値カウント: `requestAnimationFrame`ベース

### 2.2 パフォーマンス方針

- アニメーションは可能な限り `transform` と `opacity` のみで構成
- 画像は `next/image` 前提で AVIF/WebP を優先
- 動画は原則使わず、使う場合は短尺でポスター画像必須
- 初期表示は静的、インタラクションで段階的に動的化

### 2.3 アクセシビリティ方針

- `prefers-reduced-motion` を必ず尊重し、モーションを簡略化または停止
- キーボード操作でも同等に動作
- 重要情報はモーションに依存しない

## 3. モーション原則

### 3.1 トーン

静かで滑らか。速度は一定ではなく自然な減速。「安心」「清潔」「丁寧」を体感させる。

### 3.2 基本パラメータ

- 遷移時間: 180ms〜420ms
- イージング: easeOut系を標準、強いバネは禁止
- 遅延: 0ms〜120ms（情報量が多い箇所のみ段階表示）

### 3.3 基本モーションパターン

- **Fade Up**: `opacity 0→1` + `translateY 12→0`
- **Fade In**: `opacity 0→1`
- **Scale In**: `scale 0.98→1`（控えめ）
- **Hover Lift**: `translateY -2` + shadow微増（法人カードなど）
- **Focus Glow**: 入力フォーカス時に輪郭を柔らかく強調

## 4. 画面別モーション設計

### 4.1 TOP Hero

- **目的**: 先進性と安心の両立、3秒で価値理解
- **演出**:
  - 背景は静的グラデーションまたは微細ノイズ
  - メイン見出しはFade Up、サブ見出しは80ms遅延でFade Up
  - CTAはScale In（控えめ）
  - ヒーロー直下に「状態診断」カードをスッと出す（Fade In）

### 4.2 サービスプルダウン（メガメニュー）

- **目的**: 迷わせず選ばせる
- **演出**:
  - 開閉はHeightアニメではなく、`clip-path` または `scaleY` + `opacity`
  - メニュー内の各サービスは段階表示（60ms刻み）
  - モバイルはアコーディオン、アイコン回転 90deg 以内

### 4.3 状態診断（ステップUI）

- **演出**: ステップ進行バーがスムーズに進む / 選択カードは選択時に外枠が静かに強調 / 次ステップが自動スクロール（過度なジャンプ禁止）

### 4.4 料金シミュレーター

- **演出**: 合計金額はカウントアップ（0.6秒以内） / 内訳は下から展開（Fade Up） / 入力変更時は120ms程度のスムーズ更新 / 処理中はローディング必須

### 4.5 工程説明（Process Steps）

- **演出**: ステップがスクロールで順次ハイライト / 現在ステップだけやや明るく他は抑える / 工程画像はスクロールでゆっくりFade In

### 4.6 法人ページ

- **演出**: 導入モデルカードはHover Lift / 数値実績はカウントアップ / 資料DLはクリック時に控えめな成功アニメ

## 5. 画像とメディアの設計

### 5.1 画像方針

意図は必ず明確にする。装飾画像ではなく「安心と理解」を補助する画像に限定。

### 5.2 必須画像カテゴリ

- **作業環境**: 清潔な作業室、器具、保管環境
- **工程**: 異物除去、乾燥、粉骨、封入、検品
- **梱包**: 専用箱、ラベル、封印イメージ
- **人物**: 作業者の手元、制服、衛生装備
- **導線**: UIスクリーンショット

### 5.3 画像最適化ルール

- FV画像は `priority` 使用（多用禁止）
- すべて `alt` を記述
- プレースホルダーは `blur`
- 一覧は固定比率でCLP防止

---

## 6. 最先端要素の導入ポイント

### 6.1 マイクロインタラクション

ボタン押下のフィードバック / フォーム入力の成功状態 / トースト通知

### 6.2 限定的な高度演出

**以下のどれか1つに絞る（高負荷のWebGL常時描画は禁止）：**

1. トップ背景に極薄の粒子表現（Canvas）
2. 工程セクションでRiveの清潔感アイコンアニメ
3. 法人ページの数値ブロックに軽量スクロール演出

---

## 7. コンポーネント仕様（実装単位）

- `MotionProvider`: 共通duration, easing, reduced motion注入
- `RevealOnView`: IntersectionObserverトリガー
- `AnimatedNumber`: カウントアップ
- `MegaMenuMotion`: メガメニュー開閉
- `StepperMotion`: 診断・見積ステップ
- `GalleryLightbox`: 画像拡大
- `ToastFeedback`: トースト

---

## 8. 受け入れ条件

**体験品質**: ページ間でモーション統一 / 情報理解を妨げない / 過剰な揺れ・点滅なし
**パフォーマンス**: LCP 2秒目標維持 / スクロール滑らか / カクつかない
**アクセシビリティ**: `prefers-reduced-motion` 尊重 / キーボード完結

---

## 9. 実装順序

1. 共通 `MotionProvider` と `RevealOnView`
2. TOP Hero とメガメニュー
3. 診断ステッパー
4. 料金シミュレーター（数値、内訳、CTA）
5. 工程説明とギャラリー
6. 法人ページの数値と資料DL
7. 全体チューニング（reduced motion, LCP）
