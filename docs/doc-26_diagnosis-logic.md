# AI診断ロジック詳細設計書

**Project:** Ikotsu.com AI Diagnosis Logic Detailed Design
**Layer:** Functional Design
**Version:** 1.0
**Purpose:** ユーザーの不安を可視化し、最適サービスへ自動誘導する診断ロジック設計

---

## 1. 設計目的

診断はエンタメではない。**CV前の心理整理装置**。
**目的：** 不安を言語化させる / 最適サービスを提示する / 料金シミュレーターへ誘導する / 専門機関として信頼を与える

---

## 2. 診断フロー構造

**質問数：** 3〜5問以内
**完了時間：** 30秒以内
**構造：** Q1 遺骨の状態 → Q2 今後の供養予定 → Q3 保管期間 → Q4 お悩み内容（任意） → 結果表示

---

## 3. 質問設計

**Q1：現在の遺骨の状態は？**
特に問題なし / 湿気が気になる / カビがあるかもしれない / 改葬予定 / 散骨予定

**Q2：今後の予定は？**
散骨 / 墓じまい / 自宅保管 / 納骨堂 / 未定

**Q3：保管期間は？**
1年以内 / 1〜5年 / 5年以上

**Q4：不安なことは？**
費用 / カビ / 法律 / 家族説明 / 特になし

---

## 4. 判定ロジック

**ロジックはルールベース（複数条件分岐）：**

- 湿気＋5年以上保管 → 洗浄推奨
- 散骨予定 → 粉骨必須表示
- 改葬予定＋長期保管 → 洗浄＋粉骨推奨
- 未定＋不安あり → 専門相談推奨

---

## 5. 結果表示設計

- あなたにおすすめの処理
- 理由（簡潔）
- 料金目安
- 次のステップ
  **例：**「散骨予定のため、2mm以下の粉骨を推奨します。」

---

## 6. CV導線設計

結果画面下：
料金を計算する / 無料相談 / 電話する
**診断完了＝行動タイミング。**

---

## 7. アップセル誘導設計

結果に補足：
「湿気が気になる場合は洗浄もおすすめです。」
押し売りしない。提案型。

---

## 8. データ活用設計

**保存項目：** 選択履歴 / 判定結果 / 相談移行率
**将来：** BtoC傾向分析 / 法人需要分析 / コンテンツ改善

---

## 9. UI設計

カードUI / 1問1画面 / プログレスバー表示 / 戻るボタンあり / 結果は視覚的強調

---

## 10. 技術設計

Client Component / 状態管理 `useState` / ロジックは関数化 / API連携可能設計
**将来AI拡張：** OpenAI APIによる自由入力解析も拡張可能。

---

## 11. KPI

診断完了率 70%以上 / 診断後CV率 8%以上

---

## 12. 最終目標

遺骨の不安を、明確な処理提案へ変換する。
**専門機関としての信頼を構築。**
