# クロスサービス統合機能設計書

**Project:** Ikotsu.com Ecosystem Integration
**Goal:** 遺骨.comを清蓮グループのハブ化

---

## 1. 統合戦略

遺骨.comを「処理・整備の入口」と定義する。処理後の行き先を提示する構造にする。

**遺骨.comは"起点"**

| 処理 | 次の選択肢     |
| ---- | -------------- |
| 粉骨 | 保存           |
| 粉骨 | 散骨           |
| 粉骨 | 納骨           |
| 粉骨 | ダイヤモンド化 |
| 粉骨 | 墓じまい       |
| 粉骨 | 終活相談       |

---

## 2. ヘッダー統合設計

右上に「清蓮グループ」メニューを追加。ホバーで表示：

| サービス名                   | 説明                   |
| ---------------------------- | ---------------------- |
| 散骨クルーズ                 | 海洋散骨専門サービス   |
| お墓探しナビ                 | 全国墓地検索ポータル   |
| お墓じまいナビ               | 改葬・墓じまい専門支援 |
| 遺骨ダイヤモンドアドバイザー | 永遠の記念ジュエリー   |
| 終活コンシェルジュ           | 終活全般サポート       |
| 清蓮コーポレート             | 会社情報               |

---

## 3. サービス遷移ブロック設計

粉骨完了後に表示する「次に選べる供養方法」セクションを追加。

**カードUIで6分割表示。各カードに：**

- 画像
- 1行説明
- ボタン
- 想定利用者

---

## 4. 自動レコメンド機能

ユーザー選択条件に応じて関連サービスを優先表示。

| 条件            | 優先表示                 |
| --------------- | ------------------------ |
| 土付着 + 墓納骨 | お墓じまいナビ           |
| 粉骨のみ        | 散骨クルーズ             |
| 自宅供養希望    | ダイヤモンドアドバイザー |

---

## 5. 料金シミュレーター連動

粉骨シミュレーター結果画面に「次の選択肢」を表示：

- 海洋散骨の料金を見る
- 墓じまい相談をする
- ダイヤモンド化を相談

---

## 6. フッター完全統合

フッターを3カラム構造にする。

| カラム①          | カラム②              | カラム③      |
| ---------------- | -------------------- | ------------ |
| 遺骨.comサービス | 清蓮グループサービス | サポート情報 |

---

## 7. クロスSEO設計

内部リンク強化：

| ページ       | 内部リンクテキスト           |
| ------------ | ---------------------------- |
| 粉骨ページ内 | 「粉骨後に選ばれる供養方法」 |
| 散骨ページ内 | 「粉骨が必要な方はこちら」   |
| 墓じまいナビ | 「改葬後の遺骨整備はこちら」 |

GEO対策として各サービスに相互リンクを明示。

---

## 8. ビジュアル設計

**統一ブランド：**

- 白基調
- 金アクセント
- 深ネイビー補色
- 清蓮グループバッジ表示
