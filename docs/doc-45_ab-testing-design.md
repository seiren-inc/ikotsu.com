# A/Bテスト・実験設計書

**Project:** Ikotsu.com A/B Testing & Experimentation Design
**Layer:** Growth Optimization
**Version:** 1.0
**Purpose:** CV率・単価・法人比率を継続的に改善するための実験設計と運用フレームを確定する

---

## 1. 設計目的

CVは一度で決まらない。改善で作る。
**目的：** CV率を上げる / 単価を上げる / 法人問い合わせを増やす / SEOとGEO効果を伸ばす
**勘ではなく検証で勝つ。**

---

## 2. 実験対象の優先順位

- **Priority A（CV直結）：** CTA文言 / CTA位置 / 固定CTAの種類 / フォーム項目数 / 料金の見せ方
- **Priority B（単価向上）：** オプション提示方法 / 診断結果の提案文 / シミュレーターの内訳表示
- **Priority C（法人導線）：** 法人ページのファーストビュー / 資料DL導線 / 法人問い合わせボタンの配置
- **Priority D（信頼強化）：** 工程写真の出し方 / 実績表示 / 保証文言

---

## 3. 実験ユニット設計

A/Bは1回に1変数だけ変える。（CTA文言だけ / ボタン色だけ / 配置だけ）
多変数はやらない。

---

## 4. 実験フロー

1. 仮説を言語化 → 2. KPIを固定 → 3. 期間設定 → 4. 実装 → 5. 計測 → 6. 結果判定 → 7. 勝ちパターンを標準化

---

## 5. KPI設計

**主要KPI：** 問い合わせCV率 / 法人問い合わせ率 / 平均見積単価
**補助KPI：** 診断完了率 / シミュレーター完了率 / フォーム離脱率 / 電話クリック率

---

## 6. 実験テーマ一覧

| テーマ       | パターンA           | パターンB              |
| ------------ | ------------------- | ---------------------- |
| CTA文言      | 無料相談する        | 専門スタッフに相談する |
| 料金提示     | 料金表を上に        | 工程説明後に料金表     |
| 診断導線     | 診断CTAをトップのみ | 全ページ固定CTA        |
| フォーム項目 | 9項目               | 6項目                  |
| 法人ページ   | 課題訴求            | 解決策訴求             |

---

## 7. 期間設計

最低2週間。または各パターン100CV相当のデータが貯まるまで。
**少数データで判断禁止。**

---

## 8. 勝敗判定ルール

- **勝ち：** 主要KPIが5%以上改善
- **引き分け：** 差が小さい / データ不足
- **負け：** 主要KPI低下 → 即戻す

---

## 9. 運用設計

月2本実験を実施。勝ちパターンは即反映。実験ログを管理画面に保存。

---

## 10. ツール設計

Google Optimizeは終了済みのため代替。
**候補：** Vercel Web Analytics / GA4イベント比較 / Clarityの比較分析 / 将来：専用ABツール導入
初期は軽量運用。

---

## 11. リスク管理

**禁止：** SEOを崩すテスト / URL変更 / コンテンツ削除
**A/Bはページ構造を壊さない。**

---

## 12. 最終目標

CVは作業ではない。**改善文化で勝つ。**
