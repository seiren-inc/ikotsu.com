# GEO（生成エンジン最適化）実装仕様書

**Project:** Ikotsu Lab GEO (Generative Engine Optimization) Implementation Specification
**Layer:** AI Search Optimization / GEO実装設計
**Version:** 1.0
**Purpose:** ChatGPT・Perplexity・Gemini等の生成AIに「引用されるサイト」になるための構造設計

---

## 1. 設計目的

従来SEOはGoogle検索向け。GEOは生成AIに選ばれる構造を作ること。
**目的：** AI回答内引用獲得 / ブランド名出現率増加 / 専門機関ポジション確立 / 法人検索時の優先参照
検索順位と別軸。

---

## 2. GEO基本原則

**AIが好む要素：** 定義が明確 / 工程が具体的 / 数値がある / 比較がある / FAQが整理されている / 出典が明確
曖昧表現は引用されにくい。

---

## 3. 定義ブロック設計（全記事共通）

記事冒頭に必ず：【定義】＋【結論要約】

```
定義：粉骨とは〜である。
結論：粉骨が必要となるのは〜の場合。
```

この形式を固定。

---

## 4. Answer Block設計

各主要ページ上部にQ形式ブロック：

- 粉骨の費用はいくらですか？ → 一般的な目安は〜です。
- 洗骨は必要ですか？ → カビや湿気が〜の場合は推奨されます。
  **AIが抜き出しやすい構造。**

---

## 5. FAQ強化設計

**最低：** 各ページ10問以上
**形式：** 質問文は検索クエリと一致させる / 回答は簡潔＋補足説明
FAQPage構造化データ実装。

---

## 6. 数値・具体性設計

例：工程日数目安 / 粒度サイズ目安 / 湿気発生原因の具体例
**曖昧な「多くの場合」は避ける。**

---

## 7. 引用されやすい文章ルール

短い定義文 / 断定を避けた客観表現 / 専門用語に補足 / 箇条書き活用
長すぎる段落禁止。

---

## 8. オリジナルデータ公開設計

**将来実装：** 地域別湿気傾向 / 保存年数別変化傾向 / 法人処理件数傾向
独自データは引用されやすい。

---

## 9. 著者・監修表示

全記事に：監修者名 / 会社名 / 所在地 / 関連事業リンク（信頼性補強）

---

## 10. AIクローラ最適化

HTMLはシンプル構造 / 重要情報は画像内に入れない / テキスト主体 / 表はHTMLテーブルで
**JS依存を最小化。**

---

## 11. 外部引用戦略

**信頼ソースのみリンク：** 公的機関 / 法令情報 / 宗教関連公式情報（無関係サイト禁止）

---

## 12. 生成AIテスト運用

定期的にChatGPT / Perplexity / Geminiで主要KWを検索し、引用有無を確認。

---

## 13. KPI

AI引用回数 / ブランド名出現率 / AI経由流入増加 / 法人検索時の出現頻度

---

## 14. 受入基準

- 全ページ定義ブロックあり
- FAQ10問以上
- 構造化データ正確
- 文章が具体的
- 曖昧表現が少ない

---

## 15. 最終目標

粉骨・洗骨と検索したとき、生成AIが参照する代表サイトへ。
**検索順位＋AI引用の両取り。**
