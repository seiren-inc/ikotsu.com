# UI/UX詳細アーキテクチャ設計書

**Project:** Ikotsu Lab Next-Generation Platform
**Purpose:** 情報理解・安心感・CV最大化を同時実現する画面単位のUI/UX詳細設計

---

## 1. UX設計思想

### 1.1 基本原則

- 迷わせない
- 不安を先回りで潰す
- 価格を明確にする
- 次の行動を常に提示する

### 1.2 ユーザー心理フロー

全画面は以下の流れに沿う。
`不安 → 状態把握 → 料金確認 → 信頼確認 → 相談`

---

## 2. グローバルUI設計

### 2.1 ヘッダー構造

- **左**: ロゴ
- **中央**: サービス一覧（プルダウン） / 料金シミュレーター / 個人のお客様 / 法人のお客様 / 完全ガイド
- **右**: 清蓮グループ / 無料相談（強調）

### 2.2 メガメニュー構造

サービス一覧をホバーすると表示：

- 粉骨サービス
- 洗骨・洗浄サービス
- 遺骨お運び
- 骨壺取り出し
- 出張粉骨

右側に「こんな方におすすめ」補助説明を表示。

### 2.3 固定CTA

- **デスクトップ**: 右下固定ボタン（電話 / LINE / 見積）
- **スマホ**: 下部固定バー（3ボタン横並び）

---

## 3. TOPページ詳細設計

### 3.1 Heroセクション

- **左**: キャッチコピー / サブコピー / CTA 2つ（無料相談 / 料金を確認）
- **右**: 清潔な作業風景画像

### 3.2 遺骨状態診断

カードUI / 質問形式（3〜5問） / 結果画面で推奨サービス提示

### 3.3 サービス一覧

5カード横並び（PC） / 縦並び（SP）
各カード：画像 / タイトル / 1行説明 / 詳細を見る

### 3.4 料金シミュレーター簡易版

ボタンのみ表示。「最短30秒で概算」。クリックで `/simulator` へ遷移。

### 3.5 次に選べる供養方法

6カード表示：散骨クルーズ / お墓探しナビ / お墓じまいナビ / 遺骨ダイヤモンド / 終活コンシェルジュ / 清蓮コーポレート

### 3.6 信頼ブロック

実績数 / 平均対応日数 / 満足度 / 写真

### 3.7 FAQ抜粋 & 最終CTA

3〜5問表示＋詳細リンク → 背景色反転CTA（電話番号明示）

## 4. サービスページ共通UI

1. **Hero**: サービス名 / 簡潔な定義文（GEO対策） / CTA
2. **課題提示**: 「このようなお悩みはありませんか？」（アイコン＋短文）
3. **概要**: テキスト＋画像2カラム
4. **処理工程**: タイムライン形式＋画像
5. **料金表**: カード形式（サイズ別・状態別）
6. **シミュレーター導線**: 強調ブロック＋ボタン
7. **関連サービス**: 粉骨ページ下（散骨 / 墓じまい / ダイヤモンド）
8. **FAQ**: カテゴリ別
9. **固定CTA**

---

## 5. 料金シミュレーター画面設計

### 5.1 レイアウト

左：入力エリア / 右：結果表示エリア（PC）。SPでは縦積み。

### 5.2 入力UI / 結果UI

- **入力**: 状態 / サイズ / オプション / 距離スライダー
- **結果**: 合計金額大表示 / 内訳折りたたみ / 安心テキスト
- **CTA**: この内容で相談 / 正式見積依頼 / 電話相談

---

## 6. 法人ページ詳細UI

1. **Hero**: 「粉骨・洗骨の外部専門パートナー」
2. **課題提示**: 寺院・納骨堂の課題列挙
3. **提供モデル**: カード3種（完全外注 / 共同ブランド / 年間契約）
4. **処理工程**: 工程図
5. **品質管理**: 証明書 / 写真 / 衛生基準
6. **導入フロー**: 4ステップ表示
7. **専用フォーム**: 会社名 / 月間件数 / 希望内容
8. **資料ダウンロード**: フォーム入力後表示

---

## 7. 完全ガイド（SEOハブ）

1. **上部目次**: アンカーリンク
2. **定義セクション**: 粉骨とは / 洗骨とは / 改葬前処理とは
3. **比較表**: 粉骨 vs 洗骨 / 室内保管 vs 墓納骨
4. **FAQ大量配置**: 構造化データ対応

---

## 8. モバイル最適化

- カードは縦並び
- CTA常時表示
- フォームは縦フロー
- ボタンは最低44px

---

## 9. 主要UX改善ポイント

- スクロール途中でCTAを複数回提示
- 離脱しやすい料金表の下に相談導線
- FAQの最後にCTA
