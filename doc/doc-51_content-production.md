# コンテンツ制作・編集運用設計書

**Project:** Ikotsu.com Content Production & Editorial Operations Design
**Layer:** SEO Execution / Operational Design
**Version:** 1.0
**Purpose:** 粉骨・洗骨分野で検索トップを取り続けるためのコンテンツ量産・品質管理体制設計

---

## 1. 設計目的

SEOは設計だけでは勝てない。量と質の継続が必要。
**目的：** 月次コンテンツ量産 / 専門性維持 / AI引用増加 / 法人検索獲得
制作体制を固定する。

---

## 2. コンテンツ分類

- **Type A：** 商用強化記事
- **Type B：** 専門解説記事
- **Type C：** 比較・注意喚起記事
- **Type D：** 法人向け記事
- **Type E：** ローカル記事
  毎月バランスよく。

---

## 3. 月次制作目標

**最低：** 月4本 / **理想：** 月8本
| 種別 | 本数 |
|------|------|
| 専門解説 | 2 |
| 比較記事 | 1 |
| 法人記事 | 1 |
| ローカル記事 | 1 |
| FAQ強化 | 1 |
| 既存改善 | 2 |

改善も制作の一部。

---

## 4. 記事構成テンプレート

1. 結論要約 → 2. 定義説明 → 3. 背景解説 → 4. 数値・根拠 → 5. 注意点 → 6. FAQ → 7. 関連サービスCTA
   AI理解前提構造。

---

## 5. キーワード選定フロー

1. 検索ボリューム確認 → 2. 競合分析 → 3. 検索意図分類 → 4. 商用性評価 → 5. 内部リンク設計
   感覚で書かない。

---

## 6. 品質管理基準

**最低文字数：** 2,000〜3,000字
**必須要素：** 工程説明 / 数値 / FAQ / 内部リンク / 外部信頼リンク
抽象論禁止。

---

## 7. 更新設計

順位低下記事はリライト優先。
**更新箇所：** 定義強化 / FAQ追加 / 最新法解釈整理 / 内部リンク改善（放置しない）

---

## 8. 著者表示ルール

全記事に：監修者表示 / 所属明記 / 会社情報リンク（E-E-A-T強化）

---

## 9. AI最適化ルール

定義を明確に / 専門用語を説明 / 曖昧表現を減らす / 工程を箇条書きにする
**引用される文章を書く。**

---

## 10. 成果指標

自然検索流入増加 / 記事単位CV発生 / AI回答引用確認 / 滞在時間向上

---

## 11. 6ヶ月目標

主要KW 10位以内 / ロングテール占拠 / 法人記事上位表示

---

## 12. 最終目標

書き続けることで勝つ。**粉骨・洗骨分野で最も情報量のあるサイトへ。**
