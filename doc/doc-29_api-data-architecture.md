# API・データアーキテクチャ詳細設計書

**Project:** Ikotsu Lab API & Data Architecture Detailed Design
**Layer:** Functional Design
**Version:** 1.0
**Purpose:** シミュレーター・診断・フォーム・将来拡張に対応するためのAPIおよびデータ構造設計

---

## 1. 設計目的

- フロントとロジックを分離する
- 将来の法人ポータル・マイページに備える
- 分析・LTV最適化を可能にする
- 安全で拡張可能な構造を作る
  **初期は軽量。将来は拡張可能。**

---

## 2. アーキテクチャ前提

- **フロント：** Next.js App Router
- **API：** Next.js Route Handler
- **DB：** 将来Supabase / PostgreSQL拡張可能設計
  **構造：** Client → API Route → DB

---

## 3. APIエンドポイント設計

### 3.1 診断関連

`POST /api/v1/diagnosis`

- **入力：** 状態 / 供養予定 / 保管期間 / 不安内容
- **出力：** 推奨サービス / 理由 / 推奨オプション

### 3.2 シミュレーター関連

`POST /api/v1/simulator`

- **入力：** 状態 / サイズ / オプション / 地域
- **出力：** 合計金額 / 内訳 / 推奨追加
- **将来：** `GET /api/v1/simulator/history`

### 3.3 BtoC問い合わせ

`POST /api/v1/contact/b2c`

- **入力：** 氏名 / 電話 / メール / 内容 / 見積情報（任意）
- **出力：** 受付完了 / 受付ID

### 3.4 BtoB問い合わせ

`POST /api/v1/contact/b2b`

- **入力：** 法人名 / 担当者 / 件数 / 提携モデル
- **出力：** 受付完了 / 受付ID

## 4. データモデル設計（概念）

### 4.1 DiagnosisLog

id / 状態 / 供養予定 / 保管期間 / 判定結果 / 日時

### 4.2 SimulationLog

id / 入力内容 / 計算結果 / 合計金額 / 日時

### 4.3 InquiryB2C

id / 氏名 / 連絡先 / 内容 / 見積ID / 流入元 / 日時

### 4.4 InquiryB2B

id / 法人名 / 担当者 / 件数 / 希望モデル / 流入元 / 日時
