# UI仕様書マスター

**Project:** Ikotsu Lab UI Specification Master
**Layer:** Screen Design / UI仕様書
**Version:** 1.0
**Purpose:** Antigravityで再現性高く、最新アニメーション多用＋画像多用のモダンサイトを実装するための全画面UI仕様を確定する

---

## 1. UI設計の大前提

**要件：** 最新のアニメーションを多用 / 画像を複数入れ込む / モダンで高級感 / 清潔感と専門性 / 個人と法人の導線分離 / SEO/GEO最適化前提
**禁止：** 常時動く重い背景 / 過度な点滅 / 読めない装飾 / 不安を煽る演出
**アクセシビリティ：** prefers-reduced-motion対応必須 / キーボード操作可能 / コントラスト確保
**パフォーマンス：** LCP優先 / Hero画像は最適化 / アニメーションはtransform/opacity中心

---

## 2. 共通UIコンポーネント仕様

### 2.1 Header（全ページ共通）

**左：** ロゴ（Ikotsu Lab） / **中央：** ナビ / **右：** 主要CTAボタン
**ナビ項目：** サービス（プルダウン） / 法人の方へ / ガイド / 料金目安 / 会社情報 / お問い合わせ
**サービスプルダウン：** 粉骨 / 洗骨 / 出張・搬送 / お引取り / ガイド
**挙動：** スクロールで縮小 / 背景は半透明（blur） / 現在ページをアクティブ表示 / モバイルはハンバーガー
**アニメーション：** 初回表示フェードイン / スクロール縮小はスムーズ / メニュー開閉はスライド＋フェード

### 2.2 Fixed CTA（全ページ共通）

画面右下固定の縦並び：無料相談 / AI診断 / 法人相談
ホバーでラベル表示 / モバイルは横3ボタン固定

### 2.3 Footer（全ページ共通）

**セクション：** サービス / 法人 / ガイド / 法的表記 / 関連サービス / 会社情報 / 問い合わせ
**関連サービスリンク：** 散骨クルーズ / お墓探しナビ / お墓じまいナビ / 遺骨ダイヤモンドアドバイザー / 終活コンシェルジュ / 清蓮コーポレート
**最下部：** 住所 / 電話 / メール / 運営：株式会社清蓮 / 著作権表記

### 2.4 見出し・文章の表示ルール

H1はページに1つ / H2はセクションタイトル / 本文は短段落中心 / 専門用語は注釈を付ける
GEO用に各ページ上部に：定義ブロック / 要約ブロック

### 3.2 サービス一覧 `/services`

**目的：** 比較しやすく、迷わせず、詳細へ落とす
**構成：** Hero → サービス比較表（工程・納期・対象・料金目安、横スクロール対応、行ホバーハイライト） → カード一覧 → FAQ → CTA

### 3.3 粉骨 `/services/powdered`

**目的：** 粉骨の不安解消 → 相談/見積へ
**構成：** Hero（画像＋定義ブロック） → 料金ブロック（目安＋内訳思想） → 工程タイムライン（写真付き、スクロール連動） → 粒度説明（図解、ホバーで拡大） → 注意点（法的・宗教配慮） → FAQ → CTA
**画像：** 工程写真を最低6枚配置

### 3.4 洗骨 `/services/cleaning`

**目的：** カビ・湿気不安を論理で解消、洗骨の必要性を納得させる
**構成：** Hero（清潔感画像） → カビ・湿気の原因解説（図解、スクロールで段階表示） → 洗骨工程（写真ギャラリー） → 粉骨とのセット提案（押し売り禁止） → 料金目安 → FAQ → CTA

### 3.5 出張・搬送 `/services/carrying`

**目的：** 地域不安、手間不安を潰し、問い合わせへ
**構成：** Hero → 対応エリア（地図風UI、画像＋軽いホバー情報表示） → 費用目安 → 当日の流れ → 注意事項 → FAQ → CTA

### 3.6 お引取り `/services/takeout`

**目的：** 高齢者・遠方ユーザー向けの安心導線
**構成：** Hero → お引取りの条件 → 流れ → 料金目安 → FAQ → CTA

### 3.7 法人向けトップ `/for-corporate`

**目的：** 寺院・納骨堂・霊園の課題に直撃し、商談へ
**構成：** Hero（法人向け）→ 課題ブロック（湿気・保管・クレーム）→ 解決策（外注・標準化）→ 品質保証（工程公開・ログ）→ 導入フロー → 資料DL CTA（フォーム最小） → 法人問い合わせCTA
**アニメーション：** 課題→解決を左右スライドで対比演出

### 3.8 ガイドトップ `/guide`

**目的：** SEO流入の受け皿＋商用ページへ誘導
**構成：** Hero → カテゴリタブ（保存/法律/工程/トラブル/法人） → 人気記事 → 法人向け記事 → FAQへの導線

### 3.9 記事テンプレ `/guide/*`

**目的：** AI引用される記事テンプレを固定する
**構成：** 定義ブロック → 要約ブロック → 結論 → 背景 → 工程 or 仕組み → 注意点 → FAQ → 関連サービスCTA → 監修者情報 → 参考リンク
**アニメーション：** 見出し出現フェード / 図解の段階表示

### 3.10 料金目安 `/pricing`

**目的：** 価格不安の解消
**構成：** 料金の考え方 → サービス別目安 → オプション一覧 → 法人は別途案内 → FAQ
価格は「目安」表現を徹底。誤認防止。

### 3.11 お問い合わせ `/contact`

**目的：** 最短でCVを取る
**構成：** 分岐選択（個人/法人） → フォーム（ステップ式） → 確認 → 完了
必須項目最小 / 送信後の案内表示

### 3.12 会社情報 `/company`

**目的：** 信頼の担保
**構成：** 会社概要 → 所在地 → 運営方針 → 関連事業リンク → 法的表記導線

### 3.13 法的ページ `/legal/*`

**目的：** リスクゼロ運用
**構成：** 規約 / プライバシー / 特商法 / 免責 / 表現ガイド（UIはシンプル、アニメーション控えめ）

## 4. モバイルUI仕様

モバイル優先。ヘッダーは2段にしない / 固定CTAは横3ボタン / 表は横スクロール / 画像は縦スワイプギャラリー / アニメーションは軽量化

---

## 5. 画像・動画仕様

**画像：** 各サービスページ最低6枚 / トップはセクションごとに画像必須 / ガイドはアイキャッチ必須
**動画：** 原則使わない。使う場合は軽量短尺で、静止代替あり

---

## 6. アニメーション仕様（UI観点）

**必須：** セクション出現（fade/slide） / スクロール連動タイムライン / カードホバー（zoom/raise） / タブ切替（fade） / アコーディオン（smooth height）
**必須対応：** prefers-reduced-motionで無効化
**禁止：** 常時パーティクル / 常時WebGL / 背景動画ループ

---

## 7. 受入基準

- 全ページでヘッダー/フッター/固定CTAが統一
- サービスプルダウンが機能
- 法人と個人が迷わず分岐できる
- 画像が豊富で、清潔感がある
- アニメーションが上品で、軽い
- モバイルで崩れない
- Lighthouseの速度が極端に落ちない

---

## 8. Antigravityへの実装指示の要点

- ページを上記URLで生成
- 共通コンポーネントを先に作成
- 各ページのセクションを番号付きで実装
- 画像は仮素材で良いが、構造は固定
- アニメーションは軽量実装で統一
- prefers-reduced-motionを必ず入れる
