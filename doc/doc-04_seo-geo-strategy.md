## 2. SEO構造設計

### ピラーページ設計

**HUBページ：「遺骨の整備完全ガイド」**

下層ページ：

- 粉骨とは
- 洗浄とは
- 運搬の注意点
- 取り出しの流れ
- 改葬前に必要な処理

内部リンクを完全相互接続。

### 各サービスページの検索設計

**粉骨ページ例：**

- H1：`粉骨の料金と流れ｜遺骨ラボ`
- メタディスクリプション：粉骨の料金、サイズ別価格、洗浄との違いまで徹底解説。
- FAQ構造化データ実装

### ロングテール戦略

記事化対象キーワード例：

- 粉骨 何ミリ
- 粉骨 海洋散骨 用
- 遺骨 カビ どうする
- 改葬 遺骨 処理 必要か

## 3. GEO（Generative Engine Optimization）設計

生成AIに引用されるための設計。

### 必須要素

- 定義型コンテンツ
- Q&A形式明確化
- 箇条書き整理
- 数値明示
- 比較表

### 「引用されやすい文章構造」

定義文を各ページに配置する。

**例：**

> 粉骨とは、火葬後の遺骨を2mm以下のパウダー状に加工する処理を指します。

### 比較表を必ず設置

| 比較軸   | パターンA | パターンB |
| -------- | --------- | --------- |
| サービス | 粉骨      | 洗浄      |
| 保管方法 | 室内保管  | 墓納骨    |
| 搬送方法 | 出張      | 郵送      |

### 統計・実績表示

- 年間処理実績◯件
- 平均対応日数◯日
